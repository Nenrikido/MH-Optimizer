# MH Wilds Build Optimizer

React + TypeScript frontend with Flask backend for Monster Hunter Wilds build optimization.

## Project Structure

```
wilds/
├── backend/
│   ├── optimizer_app.py     # Flask API server
│   ├── opti_wilds.py        # Optimization logic
├── frontend/
│   ├── dist/                # React build output (generated by npm run build)
│   ├── public/
│   │   └── favicon.png      # Favicon
│   ├── components/          # React components
│   ├── model/               # TypeScript interfaces
│   ├── lib/                 # Utilities
│   ├── App.tsx              # Main app component
│   ├── vite.config.ts       # Vite configuration
│   └── package.json         # Frontend dependencies
└── db/
    └── ...                  # Database files
```

## API Routes

### Active Routes
- `GET /` - Serves React app (index.html)
- `GET /<path>` - Serves React app (client-side routing)
- `GET /assets/<path>` - Serves React build assets
- `POST /api/run` - Optimization endpoint (JSON format)
- `GET /api/available_items` - Returns available skills, sets, weapons

## Development Workflow

### Frontend Development

```bash
cd wilds/frontend
npm install              # Install dependencies (first time only)
npm run dev              # Development server with HMR on port 5173
npm run build            # Production build to frontend/dist/
```

### Backend Development

```bash
cd wilds/backend
python optimizer_app.py  # Starts Flask server on port 5000
```

### Access the Application

- **Development**: 
  - Frontend dev server: http://localhost:5173/ (with HMR)
  - Backend API: http://localhost:5000/api/

- **Production**: 
  - Full app: http://localhost:5000/ (Flask serves the built React app)

## Build Process

When you run `npm run build` in the frontend folder:
1. Vite builds the React app with TypeScript
2. Output goes to `frontend/dist/`
3. Flask serves files from `frontend/dist/` via relative paths (`../frontend/dist/`)
4. Public assets (favicon.png) are automatically copied from `frontend/public/`

## 🛠️ How It Works

Both optimizers use **Linear Programming** via PuLP to solve the armor optimization problem:

1. **Input:** 
   - Desired skills with weights
   - Set requirements (optional)
   - Weapon choices (optional)
   - Custom amulets (Wilds only)

2. **Optimization:**
   - Maximize: `Σ(skill_points × skill_weight)`
   - Subject to:
     - One piece per armor slot
     - Decoration slot constraints
     - Set bonus requirements
     - Skill point limits

3. **Output:**
   - Optimized armor set
   - Decoration placement
   - Achieved skill levels
   - Total score
   
## 📊 Algorithm Details

### Objective Function
```
Maximize: Σ (effective_skill_points[skill] × weight[skill])
```

### Constraints
```
1. Σ item_vars[group] = 1  for each armor group
2. decoration_slots_used ≤ decoration_slots_available
3. Σ set_pieces[set] ≥ min_pieces[set]  for required sets
4. effective_skill_points[skill] ≤ max_points[skill]
```
